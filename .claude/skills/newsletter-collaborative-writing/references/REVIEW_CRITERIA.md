# Review Criteria for Newsletter

In Step 4, perform a critical review checking these 7 criteria:

## 1. Text Preservation ✅
**Check:** User's texts have received minimal revision without altering concepts and style

**What to verify:**
- "La mia analisi" retains user's voice and core message
- Takeaways preserve user's chosen points
- Action items maintain user's intended meaning
- Only grammar, spelling, and punctuation corrections applied

## 2. Link Inclusion ✅
**Check:** All provided links are included in the final file and are active/clickable

**What to verify:**
- Every link from input appears in "I link della settimana" sections
- All markdown links are properly formatted: `[text](url)`
- URLs are clean and clickable

## 3. Narrative Flow ✅
**Check:** Concepts are connected and create an easy-to-read narrative

**What to verify:**
- Transitions between paragraphs feel natural
- Ideas flow logically from one to another
- No abrupt changes in tone or topic
- Reader can follow the story without confusion

## 4. Practical Focus ✅
**Check:** Content focuses on practical implications for enterprise software development

**What to verify:**
- Analysis connects trends to real-world development scenarios
- Enterprise context is clear and relevant
- Technical professionals find applicable insights
- Not purely theoretical or abstract

## 5. Strategic Connections ✅
**Check:** Strategic connections between trends and company moves are highlighted

**What to verify:**
- Market trends are connected to business strategies
- Company actions are analyzed in strategic context
- Broader industry patterns are identified
- "Why it matters" is clear

## 6. Actionable Insights ✅
**Check:** Concrete takeaways are provided for technical professionals

**What to verify:**
- Takeaways offer specific learning points
- Insights are applicable to AI Engineers' work
- Technical readers gain value
- Not generic statements

## 7. Action Items Quality ✅
**Check:** Action items are direct, very brief, assertive, and actionable

**What to verify:**
- Each action item starts with a verb
- Items are brief (ideally under 10 words)
- Clear and unambiguous
- Something a professional can actually do
- No vague or passive language

## Feedback Format

For each criterion not met:
```
**Criterion [N]: ⚠️ NEEDS IMPROVEMENT**

**Issue:** [Clear description of what's not working]

**Proposal:** [Specific suggestion for fixing it]

**Location:** [Section/paragraph where the issue occurs]
```

Wait for user approval before making any changes to the file.
